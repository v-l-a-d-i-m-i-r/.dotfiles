1. Install wsl following this video instruction https://www.youtube.com/watch?v=4RXDbz-EEEo
2. Reboot PC
3. Init and populate pacman keys https://gist.github.com/ld100/3376435a4bb62ca0906b0cff9de4f94b
  pacman-key --init
  pacman-key --populate
  pacman-key --refresh-keys
  pacman -Sy archlinux-keyring
4. Create user and add it to wheel group (see install.txt)
5. Init git repo in home folder and clone dotfiles.
6. Set wsl hostname https://www.srccodes.com/change-hostname-ubuntu-microsoft-windows-subsystem-for-linux-wsl-wsl2-wsl-conf-unable-resolve-hosts-name-service-not-known-list-running-shutdown-vm-srccodes/
7. Set ssh server up 

Forward port from WSL to host mashine:
1. Add firewall rule https://www.youtube.com/watch?v=7rCEG-Q0is0
2. Run in powershell as administrator
$wsl_ip = (wsl cat /etc/hostname).trim()
Write-Host "WSL Machine IP: ""$wsl_ip"""
netsh interface portproxy add v4tov4 listenport=22 connectport=22 connectaddress=$wsl_ip

Reduce vhdx size:
1. Run inside WSL
docker volume rm $(docker volume ls -q)
sudo fstrim -a
2. Run outside WSL:
wsl --shutdown
Optimize-VHD -Path "C:\Users\<USERNAME>\AppData\Local\Packages\yuk7.archwsl_35zwpb4sx6e50\LocalState\ext4.vhdx" -Mode Full
